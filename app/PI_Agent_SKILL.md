# PI Agent — 혁신과제 상세정의서 → 검토 결과서 변환

## 목적
HD현대그룹 PI 프로젝트의 **혁신과제 상세정의서(PDF, 슬라이드형)**에서
혁신과제와 세부 실행과제를 추출해, **PI 검토 결과서 Template(xlsx)**의
한 행 = 한 세부과제 형식으로 정리한다.

## 입력
1. 혁신과제 상세정의서 PDF (도메인별, 예: 생산계획(PS))
2. 검토 결과서 Template xlsx (헤더 4행, 데이터 5행부터)

## 처리 절차
1. `pdftotext -layout <pdf> out.txt` 로 텍스트 추출.
2. 다음 패턴으로 구조를 잡는다.
   - `혁신과제 리스트` 페이지 → 혁신과제명·과제오너·세부과제 목록
   - 과제별 `과제 개요` 페이지 → 과제정의·추진목적·적용사업부·기대효과
   - 과제별 `세부 실행 과제 리스트` 페이지 → 세부과제명·구현방안·유형(P/D/S/O)·검토 솔루션
   - 과제별 `세부 실행 과제 개요` / `구현 내용` 페이지 → 세부 과제정의 근거
3. 아래 매핑 규칙에 따라 세부과제별 레코드(JSON)를 만든다. 스키마는 `pi_data.json` 참조.
4. `python scripts/fill_template.py --template <Template> --data <json> --out <output>` 실행.
5. 결과 xlsx를 사용자에게 제시하고 검토 요청.

## 컬럼 매핑 규칙

| Template 컬럼 | 출처 / 규칙 |
|---|---|
| A 도메인 | 문서 도메인 고정값 (예: `생산계획(PS)`). 전체 행 세로 병합 |
| B 과제ID | 혁신과제 번호 (1,2,3…). 과제 단위 세로 병합 |
| C 과제명 | 혁신과제명. 과제 단위 세로 병합 |
| D 과제개요 | `과제 개요` 페이지의 과제정의(+추진목적 요약). 과제 단위 세로 병합 |
| E 세부과제ID | 1.1, 1.2 … |
| F 세부실행과제명 | 세부 실행 과제 리스트의 명칭 |
| G 과제정의 | 구현방안/구현내용을 **1~2문장, 명사형 종결('~한다/~함')**로 요약 |
| H 관련프로세스ID | **공란** (PDF에 ID 라벨 없음 — 추후 별도 매핑) |
| I~P 사업부 | **혁신과제 오너 기준 일괄 'O'**. 오너 없는 법인 칼럼은 공란 (아래 규칙) |
| Q 과제오너 | `적용 사업부 및 과제오너`의 오너 명단. 과제 단위 세로 병합 |
| R 시스템/솔루션 | 세부과제 리스트의 `검토 솔루션` (예: Hi-APS, OASIS, MAPS, NexFrame, APS) |
| S 기대효과-정성 | 과제 `기대 효과`의 정성 항목 요약 |
| T 기대효과-정량 | PDF에 명시된 수치(%, 시간/일 단축 등)가 있을 때만. 없으면 공란 |

### 사업부 매핑 (오너 → 컬럼)
오너 직책의 사업부명을 컬럼에 1:1 대응시킨다.
- 박상훈(조선)→조선 / 심영섭(해양)→해양 / 정우만(특수선)→특수 / 최해주(미포)→미포 / 이승환(삼호)→삼호
- 오너가 없는 `HD한조 / HHIP / HVS`는 공란 (별도 지시 시 변경)
- 모든 세부과제는 소속 혁신과제의 오너 매핑을 그대로 상속한다.

## 미정의 / 확인 필요 항목 (적용된 기본값)
- **관련프로세스 ID**: 공란 처리 (기본값). 매핑표 제공 시 H열 채움.
- **기대효과 정량(T)**: PDF에 정량 수치가 구조화돼 있지 않아 대부분 공란.
- **과제정의(G) 톤**: 명사형 종결로 통일. 다른 톤 원하면 규칙 갱신.

## 출력
- Template 서식(헤더·병합·맑은 고딕)을 유지한 채 데이터 행을 채운 xlsx.
- 새 도메인 PDF가 와도 동일 절차로 재사용 가능.
